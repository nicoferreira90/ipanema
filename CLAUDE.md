## Modules App Instructions
1. Lessons should be constructive and roughly follow the order of the particular Ramalho book chapter.
2. Modules should include between 6 and 8 lessons, keep lessons to 150 words max. Small snippets of code are acceptable in the lesson. Each lesson should be followed by a small excersize in terminal for the user, basically to see if the user understood the lesson.
3. At the end of each Module, a mini-project should be required of the user. It should take more than 12-20 lines of code, it's just to check if the user understood the content of the chapter.
4. For everything, you can use the pdf as context, but you shouldn't literally rip off examples off the text. This should be much more succinct than the book, needless to say.
5. Completing a mini-project means the Module is completed for the user and that fact should be persisted at the db level.
